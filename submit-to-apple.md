# Submit Ruck & Run to the App Store

## Pre-Submission Checklist

### App Store Connect — Metadata
- [ ] App name: **Ruck & Run**
- [ ] Subtitle (30 chars max): e.g. "Track runs, walks & rucks"
- [ ] Description — **first paragraph must state: this app does not sync with Apple Health.** Be direct, not apologetic. Example: "Ruck & Run does not integrate with Apple Health or HealthKit. All data is stored on-device only. If Apple Health sync is a must-have for you, this app isn't for you."
- [ ] Keywords (100 chars max): think run tracker, ruck, walk tracker, GPS, no subscription, simple
- [ ] Support URL: https://jwatsondev.github.io/simple-run
- [ ] Privacy Policy URL: https://jwatsondev.github.io/simple-run/privacy
- [ ] Category: Health & Fitness (primary), Sports (secondary)
- [ ] Age rating: complete the questionnaire (should be 4+)

### Screenshots
- [ ] iPhone 6.9" screenshots (required) — at least 1, up to 10
- [ ] iPhone 6.5" screenshots (optional but recommended)
- [ ] iPad screenshots (only if you support iPad)

### App Version Info
- [ ] Version: 1.0.0
- [ ] Build: 12 (already on TestFlight — confirm it appears in App Store Connect)
- [ ] What's New (first release — can leave blank or write a short line)

### Review Information
- [ ] Sign-in required? No
- [ ] Demo account? N/A
- [ ] Notes for reviewer: explain background location use ("App tracks GPS during active runs/rucks/walks. Background location is required to continue tracking when the screen locks.")

### Legal / Compliance
- [ ] Export compliance: answer the encryption questions (standard HTTPS = yes to encryption, no to exemption — mark as standard)
- [ ] Privacy labels: fill out data types collected. This app stores data on-device only — should be minimal. Select "Data Not Collected" if accurate.

### Final Checks
- [ ] Build passes Apple's automated checks in App Store Connect (no rejections flagged)
- [ ] No placeholder text or lorem ipsum anywhere in the app
- [ ] App icon looks correct at all sizes in App Store Connect preview
- [ ] TestFlight build selected is Build 12

## Submitting
1. In App Store Connect, go to your app → press **+** next to iOS App to create a new version (1.0.0)
2. Fill in all metadata above
3. Select Build 12 from TestFlight
4. Submit for Review

## After Submission
- Apple review typically takes 24–48 hours for a first submission
- Watch for emails from App Store Connect — they'll notify you of approval or rejection
- If rejected, read the rejection reason carefully before resubmitting — do not resubmit without addressing it
